# 🏥 AIoT 專家知識庫：知識條目生成工具使用指南

## 使用[AIoT知識條目生成器界面介紹影片](https://youtu.be/tMkqqWCIlro)可參閱

## 1. 專案簡介與目的
本工具旨在協助藥師與臨床醫療人員，將臨床指引 (Guidelines)、標準作業程序 (SOP) 或教科書內容，利用 AI 自動轉換為符合 RAG (檢索增強生成) 規範的結構化資料。

**核心目標**：建立高品質、內容一致、易於檢索的專家知識庫，以輔助臨床決策 。

---

## 2. 快速開始 (Quick Start)

### 步驟一：取得 Google Gemini API Key
本工具使用 Google Gemini 模型進行分析，您需要先申請免費的 API Key。

1. 前往 [Google AI Studio](https://aistudio.google.com/app/apikey)。
2. 登入您的 Google 帳號。
3. 點擊 **"Create API Key"**。
4. 複製產生的 Key（以 `AIza` 開頭的字串）。
5. 回到本工具網頁，貼上 Key 即可開始使用。

### 步驟二：使用生成工具
1. **上傳檔案**：支援 PDF 格式（可一次上傳多個檔案）。
2. **補充指令**：若有特殊需求，可在文字框輸入（例如：「請加強副作用的描述」）。
3. **生成條目**：點擊按鈕，AI 將自動解析並生成 Markdown 格式的知識條目。
4. **複製結果**：點擊「複製」按鈕，將內容貼入 Excel 的 `content` 欄位。

---

## 3. 知識條目製作規範 (重要！)

為了確保 AI 能夠準確檢索，請務必遵守以下的格式規範。

### 3.1 Excel 檔案結構
每個單位需繳交 Excel 檔案 (`.xlsx`)，檔案內**僅包含一個工作表**，且**只有兩欄** ：

| 欄位名稱 | 說明 |
| :--- | :--- |
| **title** | 知識條目的標題與來源描述（必填）。這是 AI 檢索的座標。 |
| **content** | 知識條目的完整內容（必填）。使用 Markdown 語法排版，嚴禁包含個資。 |

> **⚠️ 檔案命名規則**：`{單位縮寫}_{主題}_{YYYYMMDD}_{版本}.xlsx`
> 例如：`PHARM_Vanco_TDM_20250827_v1.xlsx`

### 3.2 Title (標題) 命名規則
標題必須讓人或系統「一眼看懂座標」。請根據文件性質，從以下三種模式擇一使用，並使用 `|` 符號分隔 。

#### 🔹 模式 A：來源導向 (適用指引/教科書)
* **格式**：`來源文件全名 | 章節 | 小節 | 主題關鍵詞 | 版本`
* **範例**：`NCCN v2024 Colon Cancer | Surveillance | Imaging Follow-up | CT schedule | 版本2024.1`

#### 🔹 模式 B：臨床情境導向 (適用 SOP/流程)
* **格式**：`主題關鍵詞 | 臨床情境 | 來源/SOP | 版本`
* **範例**：`高血鉀急症處置 | 急診流程 | 院內SOP v2`
* **範例**：`Vancomycin TDM 抽血時間點判讀 | 內科住院 | 院內SOP v3`

#### 🔹 模式 C：主題導向
* **格式**：`核心主題 | 子題 | 資料日期`
* **範例**：`口服抗凝劑(DOAC)術前停藥 | 手術風險分級 | 2025-07-30`

### 3.3 Content (內容) 撰寫原則
* **不拆分原則 (Consolidation)**：一個檔案或一個完整章節應生成**單一條**知識條目。避免將步驟拆得太碎，以免 AI 斷章取義 。
* **結構化**：使用 Markdown (`#`, `##`, `-`, `**`) 讓內容層級分明。
***引用來源**：Content 文末需加上 `*參考資料：[文件名稱] ([版本/日期])*`。

## 4. 特殊內容處理：表格與流程圖 🖼️
**注意**：知識庫的 `content` 欄位只能儲存文字，**無法儲存圖片**。若原始文件包含重要的表格或流程圖，請依照以下標準作業程序轉換。

### 處理流程
1.  **螢幕截圖**：擷取清晰完整的表格或流程圖（包含標題與註解），存為 PNG/JPG。
2.  **使用 AI 工具轉換**：
    * **表格 (Tables)**：建議使用 [Table Extractor GPT](https://chatgpt.com/g/g-688a876e8e6481919706e94b0b3cdc3d-table-extractor)。它能將圖片表格轉為 Markdown 或 CSV。
    * **流程圖 (Flowcharts)**：建議使用 [Flowchart Extractor GPT](https://chatgpt.com/g/g-68ae95df59a481918462f4880eda9635-flowchart-extractor)。它能解析路徑與決策點。
    * **通用工具**：若無法使用上述工具，可使用 ChatGPT-4o、Claude 3.5 Sonnet 或 Gemini 1.5 Pro，上傳圖片並搭配下方提示詞。

### 轉換專用提示詞 (Prompt)
請複製以下方框內的文字，貼給 AI 工具進行轉換：

```text
請將此圖片中的表格/流程圖，轉換為結構化的文字格式。請遵循以下要求：
1. 保持原有的階層結構與邏輯關係。
2. 使用 Markdown 格式呈現，特別是表格語法。
3. 確保所有數據、標籤、流程路徑都完整保留。
4. 對於流程圖，請用文字描述每個步驟的決策點與後續動作。
5. 輸出結果需為台灣繁體中文。
